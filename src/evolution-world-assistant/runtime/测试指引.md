- [x] SillyTavern 已启动

## 二、Prompt 预览

### 基本预览

1. 在 Prompt 预览区，选择目标工作流（或「自动选择」）
2. 点击 **📝 生成预览**
3. 检查：

| # | 检查项 | 预期 | ✅/❌ |
|---|--------|------|-------|
| 5 | 消息列表出现 | 显示 N 条消息卡片 | |
| 6 | 每条消息有 role 徽章 | `SYSTEM` 紫色、`USER` 绿色、`ASSISTANT` 橙色 | |
| 7 | 消息可展开/折叠 | 点击 header 切换 | |
| 8 | 消息数量显示 | 顶部显示「共 X 条消息」 | |

### EW 条目名标签

> 这是本次核心验证项。

9. 在展开的消息内容中搜索 `[EW/` 或 `[Controller]` 或 `[Dyn]`：
   - **Controller 标签**：应出现 `[EW/Controller]`（或你自定义的控制器名）
   - **Dyn 标签**：应出现 `[EW/Dyn/xxx]` 格式，`xxx` 是条目名
   - 标签文字应**高亮显示**（紫色背景）

| # | 检查项 | 预期 | ✅/❌ |
|---|--------|------|-------|
| 9 | Controller 内容前有 `[EW/Controller]` | 高亮标签可见 | |
| 10 | 至少一个 Dyn 条目前有 `[EW/Dyn/xxx]` | 高亮标签可见 | |
| 11 | 标签名与世界书条目名一致 | 对照世界书验证 | |

**快速验证方法**：展开含 worldInfoBefore 的消息 → Ctrl+F 搜 `[EW`。

## 三、快照检视

1. 点击 **📸 读取快照**
2. 检查：

| # | 检查项 | 预期 | ✅/❌ |
|---|--------|------|-------|
| 12 | Controller 状态 | 显示 ✅ 有内容（如果有活跃控制器）| |
| 13 | Controller 内容 | 显示 EJS 代码摘要（或完整展开）| |
| 14 | Dyn 表格 | 列出所有 Dyn 条目：绿/红圆点 + 名称 + 摘要 | |
| 15 | enabled 状态正确 | 绿点 = 启用，红点 = 禁用 | |

**对照验证**：打开 ST 世界书编辑器，对比 EW/Dyn/* 条目的数量和内容是否一致。

## 四、运行日志

### 触发一次工作流

1. 在调试 tab 「手动运行输入」框留空（使用最新楼层）
2. 点击 **▶ 手动运行**
3. 等待完成，检查运行记录区：

| # | 检查项 | 预期 | ✅/❌ |
|---|--------|------|-------|
| 16 | 状态徽章 | ✅ 成功（绿）或 ❌ 失败（红）| |
| 17 | 元信息 | 显示「手动 · N 工作流 · Xms」 | |
| 18 | 时间戳 | 显示当前时间 | |
| 19 | 失败原因 | 如失败，红色错误块显示原因 | |

### Flow IO 详情

| # | 检查项 | 预期 | ✅/❌ |
|---|--------|------|-------|
| 20 | Flow 卡片 | 每个执行的工作流一张卡片 | |
| 21 | 展开/折叠 | 点击 header 可展开 Request/Response | |
| 22 | Request 内容 | 显示请求 JSON 摘要 | |
| 23 | Response 内容 | 显示 AI 返回 JSON 摘要 | |

## 六、条目名注入端到端验证

> 这是完整流程的 E2E 测试，验证条目名最终出现在 AI 请求中。

1. 确保角色卡有活跃的 Controller + ≥1 个 Dyn 条目
2. 正常发送一条消息（不是手动运行）
3. 打开浏览器 F12 → Network → 过滤 `generate`
4. 找到发往 AI 的请求 → 查看 messages 数组
5. 在 worldInfoBefore/After 对应的 message.content 中：
   - **应看到** `[EW/Controller]` 在控制器内容前
   - **应看到** `[EW/Dyn/xxx]` 在各 Dyn 条目内容前

或者使用调试面板：
1. 调试 Tab → 生成 Prompt 预览
2. 展开含世界信息的消息
3. 确认标签存在且高亮
