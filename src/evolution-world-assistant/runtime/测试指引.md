# Evolution World 调试面板 — 测试指引

## 前置条件

- [x] SillyTavern 已启动
- [x] Evolution World 插件已加载（控制台无报错）
- [x] 至少有一张角色卡绑定了含 `EW/Controller` + `EW/Dyn/*` 条目的世界书
- [x] 至少配置了一个 API 预设（酒馆连接器或自定义 API）

> [!NOTE]
> ST-Prompt-Template 不再是硬性前置条件。EW 内置了自己的 EJS 引擎（`ejs-internal.ts`），
> 可独立完成 Controller EJS 渲染和世界书条目 EJS 执行。
> 但酒馆主生成仍由酒馆自身的 EJS 引擎渲染 Controller 条目。

## 六、Controller 渲染端到端验证

> 验证 controller_model JSON → EJS 编译 → 写入世界书 → 酒馆主生成渲染的完整链路。

1. 确保工作流 AI 返回了 `controller_model`（含 `fallback_entries` 和/或 `char_detection`）
2. 检查 ST 世界书编辑器中 `EW/Controller` 的内容：
   - 应为自动编译的 EJS 代码（`<% for (const _ewName of ...) { %>` 等）
   - **不是** AI 直接写的（AI 只返回 JSON，插件编译成 EJS）
3. Dyn 条目应全部为**红灯**（禁用），内容正确
4. 正常发送一条消息，F12 Network 检查 AI 请求：
   - messages 中应看到 Dyn 条目的**渲染后内容**（由酒馆 EJS 引擎通过 Controller 的 getwi 拉取）
